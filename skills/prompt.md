# Prompt Creation

Prompt 把章纲中的故事任务转成 writer 可以直接展开的单章行动过程。每章拥有一个独立 Prompt；创建任务以一幕为单位，长幕以连续叙事批次为单位。

## 任务范围

普通长度的幕由一个 prompt-crafter 顺序创建该幕全部 Prompt。它先理解整幕如何从 `start_state` 走向 `end_state`，再逐章形成独立文件。

长幕按人物目标、关系、信息、资源状态、重大转折和唯一事件的稳定交接划分连续批次。一个 prompt-crafter 处理一个批次中的多章，批次之间按幕内顺序推进。3–5 章可以帮助安排调度，但不决定批次边界。

任务范围与产物粒度相互独立：

```text
创建任务：一幕或一个连续批次
最终产物：一章一个 prompts/vol-N-ch-M.md
```

## 创作上下文

prompt-crafter 从当前幕纲、任务范围内的章纲、批次入口和出口、必要人物设定、项目文风、题材、有效正文入口和少量相关知识建立幕内理解。

`settings/writing-style.md` 与 `settings/genre-setting.md` 提供本项目的表达方向和题材期待。创建者把当前章节真正需要的内容转化为 Prompt 中的正向写作材料，使 writer 不必再回到这些来源寻找上下文。

## 单章 Prompt

每份 Prompt 独立写入 `prompts/vol-N-ch-M.md`：

```markdown
---
prompt_contract: 2
chapter: vol-1-ch-1
act: vol-1-act-1
target_chars: 4000
min_chars: 3200
---

## 本章要完成的变化
- 幕内作用：{本章在当前幕中的作用}
- 核心变化：{人物、关系、信息或局势的不可撤回变化}
- 读者期待：{本章回应、加深或转移的问题}
- POV：{人物与限知边界}

## 承接现场
- 时间与地点：{具体入口}
- 上一章留下的动作或状态：{本章从哪里开始}
- 当前压力：{人物为什么必须行动}

## 人物发动机
- {人物}：眼前目标；已知、未知或误判；可用筹码；不能退让的理由；行动与说话方式

## 场景推进

### 场景：{名称}｜权重：核心/承接/转折
- 入场状态：{人物、地点和正在发生的动作}
- 目标与阻力：{人物想取得什么；谁或什么阻止他}
- 行动与反制：{双方怎样试探、施压、交换、追逐或调整}
- 转折与选择：{什么改变判断；人物作出什么有代价的选择}
- 可见结果：{关系、信息、资源、风险或局势怎样变化}
- 下一步触发：{结果怎样逼出下一场或下一章}
- 展开重点：{最值得写足的博弈、情绪或身体经验}

## 必守事实与边界
- 必须发生：{通过人物行动交付的变化}
- 相关事实：{当前章真正需要的身份、能力、资源和信息阶段}
- 保留边界：{会影响后续承接的事实范围}

## 收束画面
{具体动作、证据、选择或新状态}

## 本章质感
- {当前章节有效的文风、题材和叙事执行提示}
```

Prompt 通过人物行动组织内容。事件名称、结果清单、方法标签和抽象主题不能代替人物目标、阻力、反制、选择与后果。

`target_chars` 和 `min_chars` 帮助分配场景展开空间。它们服务于轻重安排，不承担文学判断。

## 幕内连续性

prompt-crafter 按章节顺序创建文件。每章的收束与下一章入口共同来自幕纲和章纲；批次结束时确认出口能够交给下一批次继续创建。

创建者遇到幕纲和章纲无法共同成立的情况时，把冲突和受影响范围返回顶层。Prompt 不替规划层改写上位事实。

## 完成语义

当前幕或 batch 完成后，顶层继续长期目标范围内的下一段 Prompt 创建。长期 `range` 内每章的 Prompt 全部创建并落盘后，cursor 才进入 `prompts.ready`；这个节点表示所选写作范围已经可以整体进入 Fast 或 Full。

## 显式 Prompt 审查

只有用户明确提出审核提示词时，顶层创建 prompt-reviewer，并建立临时 `prompt.review` 任务。

reviewer 先单独阅读目标 Prompt，判断它自身能否形成清楚、可执行的人物行动；随后只增加所在幕纲和对应章纲，核对 Prompt 与规划的关系。审查结果写入当前 task，不改变默认 Prompt 创建链。
